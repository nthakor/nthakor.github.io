0. Notion basics you need to know first

If you've never used Notion seriously, read this section. Every step below assumes these moves.

The slash menu /

Type a forward slash anywhere on a blank line and a menu pops up. This is how you add everything in Notion — databases, headings, callouts, quotes, dividers, toggles. You'll see commands like:

  • /database - inline — create a database embedded on the current page
  • /callout — a colored box with an emoji
  • /heading 2 or just type ## then a space — a large section heading
  • /divider — a thin horizontal line
  • /todo — a checkbox list item
  • /quote — an italicized quote block
  • /toggle heading 2 — a collapsible section

How to add a database property

Open any database → click the + at the end of the column row → a panel opens asking for property name and type. Pick the type (Text / Select / Date / Relation / etc.), name it, hit Enter.

Property types you'll use

Titleauto-createdThe main name of the row. Every database has exactly one.
Textfree-formPlain text, any length.
SelectdropdownPick one option from a fixed list. You define the options.
Multi-selecttagsPick multiple options from a list.
DatecalendarA single date or a date range.
EmailclickableValidated email address.
RelationlinkLink to another database. This is the magic.

What a "relation" actually does

A relation is a two-way link between two databases. If you add a relation from Tasks → Projects, then: (1) each Task row gets a "Project" field where you pick a project, and (2) each Project row automatically shows a "Tasks" list of everything linked to it. You fill one side, both sides update.

How to create a view

At the top of any database, next to the view name, click the + to add a new view. Pick a type (Table / Board / Calendar / List / Timeline / Gallery). Then click the view name → Filter or Sort to configure it.

Duplicating a row (how "templates" work in this system)

Right-click any database row → Duplicate. A copy appears. Rename it. That's how you'll use the 📌 template rows below — duplicate, rename, fill in.

3. Build order — do it in this exact sequence

Relations require their target database to already exist. If you try to add a "Project" relation to Tasks before you've created the Projects database, Notion will have nothing to link to. Follow this order:

  1. Hub page
  2. People database (no relations)
  3. Projects database (relation → People)
  4. Meetings database (relation → People, Projects)
  5. Tasks database (relation → People, Projects, Meetings) ← the linchpin
  6. Views
  7. Input templates (the 📌 rows)
  8. Meeting templates
  9. Home page layout
  10. Setup & Guides subpage

Step 1 — Create the hub page

1

Create a new page

In Notion's left sidebar, click + Add a page. A blank page opens.
2

Give it a title

Type: Productivity OS. Press Enter.
3

Add an icon

Hover just above the title → click Add icon → pick 🧠 (or any single emoji you like).
4

Add a cover

Hover above the title → click Add cover → click the cover → Change cover → Unsplash tab → search "mountain", "fog", or "sunrise" → pick one → Reposition if needed.

Step 2 — Build the People database

1

Insert a database

On a blank line in the hub page, type /database and pick Database - Inline. A small table appears.
2

Name it

At the top of the new database, click the "Untitled" header → type People → press Enter.
3

Rename the default "Name" title column

It's probably already called "Name". If not, click the column header → Rename → type Name.
4

Add the Relationship select property

Click the + at the end of the column row → type Relationship → pick Select type → click Done. Now click any cell in the Relationship column → type each option one at a time and press Enter: Manager, Direct Report, Peer, Stakeholder, External.
5

Add Role, Team, Email

Click + again → Role → Text. Repeat for Team → Text, and Email → Email.

Step 3 — Build the Projects database

1

Insert and name

Below the People database, add a blank line → /database → Database - Inline → name it Projects. Rename the title column to Project.
2

Add Status select

+ column → Status → Select. Options: Backlog, Planning, In Progress, Blocked, Done, Archived.
3

Add Priority select

+ column → Priority → Select. Options: P0, P1, P2, P3.
4

Add Goal, Start, Target

Goal → Text. Start → Date. Target → Date.
5

Add the Stakeholders relation (first relation!)

+ column → type Stakeholders → pick Relation type → a panel slides in asking which database to link to → pick People → make sure Show on People is toggled ON (this creates the reverse link so each person shows their projects). Click Add relation.

Step 4 — Build the Meetings database

1

Insert and name

/database → Inline → name it Meetings & 1:1s. Rename title column to Title.
2

Add Type select

Type → Select. Options: 1:1 Manager, 1:1 Direct Report, Project Meeting, Team Sync, Stakeholder, Other.
3

Add Date and Summary

Date → Date. Summary → Text.
4

Add Attendees relation → People

Attendees → Relation → target: People → Show on People: ON → Add relation.
5

Add Project relation → Projects

Project → Relation → target: Projects → Show on Projects: ON → Add relation.

Step 5 — Build the Tasks database (the linchpin)

This is the most important step. Tasks has three relations, one to each other database. Together they're what makes the whole system work.
1

Insert and name

/database → Inline → name it Tasks. Rename title column to Task.
2

Add Status select

Status → Select. Options: Inbox, Next, In Progress, Waiting, Done.
3

Add Priority, Do Date, Due

Priority → Select → P0/P1/P2/P3. Do Date → Date. Due → Date.
4

Add Assignee → People

Assignee → Relation → target People → Show on People: ON.
5

Add Project → Projects

Project → Relation → target Projects → Show on Projects: ON.
6

Add Meeting → Meetings (the linchpin)

Meeting → Relation → target Meetings & 1:1s → Show on Meetings: ON. This is the one that makes meeting-note → task flow work.

Step 6 — Create the views

In each database, click the + next to the view name to add a new view. Here's every view to create:

On the Tasks database

1

Today (table)

+ view → pick Table → name Today → click the view name → Filter → Status → is not → Done. Then Sort → Due → Ascending.
2

Kanban (board)

+ view → Board → name Kanban → "Group by" → Status.
3

Inbox to Process (table)

+ view → Table → name Inbox to Process → Filter → Status → is → Inbox.
4

Waiting On (table)

+ view → Table → name Waiting On → Filter → Status → is → Waiting.

On the Projects database

5

Active Projects (board)

+ view → Board → name Active Projects → Group by → Status.

On the Meetings database

6

Calendar

+ view → Calendar → name Calendar → Calendar by → Date.
7

By Type (board)

+ view → Board → name By Type → Group by → Type.
8

1:1 Log by Person (list)

+ view → List → name 1:1 Log by Person. Filter: Type → is → 1:1 Manager, then click + Add filter → OR → Type → is → 1:1 Direct Report. Then click the view name → Group → Attendees. Finally Sort → Date → Descending. This gives you a rolling log of every 1:1 grouped by person — the one view to use before any 1:1.

Daily use — the one habit to build

1

Before each meeting

Open the Meetings database → click + New → in the dropdown that appears, click the 📌 template row you want → Notion opens a fresh page pre-filled with the template. Set the Date and Attendees at the top.
2

During the meeting — the only move that matters

Type notes in the template sections. When an action item comes up, don't just write it as a bullet. Instead:
  1. Scroll to the top of the meeting page
  2. Find the Tasks property
  3. Click + New — a blank task row opens
  4. Type the task, set Assignee + Due
The Meeting link fills in automatically because you created the task from this page. That single click is the entire system.
3

That's it

The task now shows up in Tasks → Today, on the Project page, on the Person's People page, and here. You typed it once.

Optional extras (add later if you want)

Areas database — ongoing responsibilities like "Hiring", "Team Health", "On-call". Add a relation from Projects → Areas if you want to group projects by long-term theme. Skip on day one.

Notes & Knowledge database — reference material like RFCs, decisions, onboarding docs that aren't tied to a single meeting. Add a relation from Projects → Notes. Skip on day one.

That's the whole system. Four databases, eight views, four input templates, three meeting templates, one minimal home page, one subpage for guides, one habit (create tasks from inside meeting pages). Take your time — it's a 2-hour build if you're careful, 30 minutes if you're fast.
