🧠 Nilay's Productivity OS

A hybrid PARA + GTD system in Notion. Capture once, surface everywhere.

1. The system at a glance

Six databases, connected by relations. The arrows below show how a single piece of information flows through the system without ever being typed twice.

The one rule that makes the whole thing work: you only ever type something once, and that's almost always inside a Meeting note. Everything else is a relation that surfaces it where you need it.

2. The flow — what happens when you take a note

Step 1. You're in a 1:1 with your manager. You open a new page from the Meetings database, set Date + Attendees = your manager.
Step 2. You type during the meeting:
- [ ] Draft Q3 staffing plan — Friday
Step 3. After the meeting (literally 10 seconds), you select that line → "Turn into → Page in" → pick the Tasks database. Set Project = "Q3 Planning", Assignee = you.
Step 4. That single keystroke now appears in:
  • Today view of Tasks (because Status = Next, Due = Friday)
  • The Q3 Planning project page (rollup of all related tasks)
  • Your manager's People page (rollup of every task born in your 1:1s)
  • The 1:1 note itself (still linked back via the Meeting relation)
You wrote it once. It now lives in four places. That's the entire point.

4. Why PARA + GTD (and not something else)

ModelStrengthWhy it's not enough on its own for you
Pure GTDDaily execution, "next action" disciplineNo home for ongoing manager responsibilities (your team, hiring). Weak on reference material.
Pure PARABeautiful information architectureNo daily execution loop. You'll know where everything is but not what to do next.
Bullet Journal / daily notesFast captureNothing connects. You re-write the same action item in 3 places — exactly what you said you want to avoid.
Second Brain / ZettelkastenDeep knowledge workOverkill for an EM whose unit of work is meetings → decisions → actions, not essays.
OKRsGoal alignmentHeavy ceremony for individual use. Project goals + Areas cover the same intent.
Eisenhower MatrixPrioritizationIt's a lens, not a system. Baked in via Priority P0–P3.

The hybrid wins because: your day is 60–70% meetings, so the system has to be meeting-centric on input and project-centric on output. PARA gives you the buckets; GTD gives you the loop; the relations between databases are what kill the double-write problem.

6. How to recreate this in your work Notion — step by step

Build the databases in this exact order. Order matters because later databases reference earlier ones via relations.

Step 1 — Create the hub page

  1. In your Notion sidebar, click + New page. Title it Nilay's Productivity OS. Add an icon (🧠).
  2. This page will be the parent for all 6 databases.

Step 2 — Create People (no relations yet)

  1. On the hub page, type /database → Database — Full page. Title: People.
  2. Add properties:
    • Name (Title — already exists)
    • Role (Text)
    • Relationship (Select: Manager, Direct Report, Peer, Stakeholder, External)
    • Team (Text)
    • Email (Email)
    • Notes (Text)

Step 3 — Create Areas

  1. Back on the hub page, /database → Areas.
  2. Properties: Area (Title), Type (Select: Responsibility, Goal, Resource), Description (Text).

Step 4 — Create Projects

  1. /database → Projects.
  2. Properties: Project (Title), Status (Select: Backlog, Planning, In Progress, Blocked, Done, Archived), Priority (Select: P0, P1, P2, P3), Start (Date), Target (Date), Goal (Text), PRJ (Unique ID, prefix PRJ).
  3. Add relations:
    • Area → relates to Areas DB (turn on "show on Areas")
    • Stakeholders → relates to People DB (turn on "show on People")

Step 5 — Create Meetings & 1:1s

  1. /database → Meetings & 1:1s.
  2. Properties: Title, Date, Type (Select: 1:1 Manager, 1:1 Direct Report, Project Meeting, Team Sync, Stakeholder, Other), Summary (Text).
  3. Relations:
    • Attendees → People (two-way)
    • Project → Projects (two-way)

Step 6 — Create Tasks (the linchpin)

  1. /database → Tasks.
  2. Properties: Task, Status (Select: Inbox, Next, In Progress, Waiting, Done), Priority (P0–P3), Due (Date), Do Date (Date), Context (Multi-select: @work, @home, @deep, @quick, @waiting), TSK (Unique ID).
  3. Relations (this is where the magic happens):
    • Project → Projects (two-way)
    • Meeting → Meetings & 1:1s (two-way) ← this is the most important relation in the entire system
    • Assignee → People (two-way)

Step 7 — Create Notes & Knowledge

  1. /database → Notes & Knowledge.
  2. Properties: Title, Type (Idea, Reference, Decision, Learning, Quick Note), Tags (Multi-select), Created (Created time).
  3. Relations: Project → Projects, Meeting → Meetings (both two-way).

Step 9 — Create real Notion templates inside Meetings

This is the part the API can't do, but you can in 5 minutes:

  1. Open the Meetings & 1:1s database.
  2. Click the ▾ next to the blue New button → + New template.
  3. Create three: 1:1 with Manager, 1:1 with Direct Report, Project Meeting.
  4. For each template, paste the section structure from the next part of this doc.
  5. Set the Type property in each template so it auto-fills.

8. Daily & weekly rituals (the part that makes it actually stick)

Morning (3 min): Open Tasks → Today view. Pick your 3 P0/P1s for the day. That's it.
After every meeting (1 min): Open the meeting note. For each - [ ] action item, highlight the line → "Turn into → Page in" → Tasks. Set Project + Assignee. Done. Notion auto-links the Task back to the meeting via the relation.
Friday weekly review (15 min):
  1. Tasks → Inbox to Process view. For each loose task, assign Project + Status + Due.
  2. Projects → Active Projects board. Move cards. Update statuses.
  3. Tasks → Waiting On view. Nudge whoever's blocking you.
  4. Open your manager's People page. Skim open items. Seed your next 1:1 agenda.

9. Pro tips

  • The Meeting → Task relation is the single most important one in the system. If you only get one thing right, get this. It's what eliminates double-writing.
  • Don't over-classify. Areas should be 5–7 max. Projects rarely more than 10 active.
  • Use Context tags (@deep, @quick) to time-block. "I have 30 min before my next meeting" → filter Today by @quick.
  • Your manager's People page is your secret weapon. Open it 60 seconds before every 1:1. You'll look organized because you actually are.
  • The Inbox status is sacred. When something pops into your head mid-meeting, just dump it as a Task with Status = Inbox. Process it Friday.

10. Where to start (your first 30 minutes)

  1. 10 min: Fill People — your manager, every direct report, your skip-level, top 3–5 stakeholders.
  2. 10 min: Create Areas (5–7 max). For an Affirm EM these might be: Team Leadership, Hiring & Recruiting, [Your team]'s Roadmap, Cross-functional Partnerships, Personal Growth.
  3. 5 min: Add 3–5 current Projects under those Areas.
  4. 5 min: Use a meeting template at your next 1:1. Convert one action item to a Task. Watch it appear on the assignee's People page.

— That's the whole system. The rest is just discipline.
