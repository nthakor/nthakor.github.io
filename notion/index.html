1. Why this system

You wear three hats every day — engineering manager (1:1s, hiring, growth), project lead (technical work, decisions, RFCs), and individual (your own todos and notes). The single biggest productivity tax for managers is writing the same thing in multiple places: meeting notes in one doc, action items in another, project status in a third, and 1:1 themes in a fourth.

The fix is a relational database setup where each piece of information lives in exactly one row, but surfaces in every relevant view automatically. PARA gives you the categorization (People / Areas / Projects), GTD gives you the action discipline (Inbox → Next → Waiting → Done), and Notion's relations stitch them together.

The core insight: meeting-centric input, project-centric output. You write notes during meetings (where your attention is) but you review them by project (where your work is). The relations make this free.

5. The daily flow + worked example

Here's exactly how it played out with the dummy data already in your workspace:

1You sit down for a 1:1 with Alex. Open Meetings DB → New → pick the 1:1 Direct Report template. Set Title = "1:1 with Alex — Apr 6", Date = today, Attendees = Alex, Project = Checkout Latency Reduction. Type notes into the template's pre-filled headings.
2Action item comes up: "Review Alex's caching RFC by Apr 10." You go to the Tasks property at the top of this meeting → New row → Task = "Review Alex's caching RFC", Status = Next, Priority = P0, Due = Apr 10, Assignee = Alex, Project = Checkout Latency Reduction. The Meeting relation auto-fills because you created it from the meeting page.
3You stop typing. Watch what just happened:
  • The task appears in Tasks → Today (because Status ≠ Done)
  • It appears on the Checkout Latency Reduction project page (Tasks relation)
  • It appears on Alex Rivera's People page (Tasks relation, Assignee)
  • It appears on the 1:1 with Alex — Apr 6 meeting page (Tasks relation)
  • You typed it once.
4Friday review. Open Tasks → Inbox to Process → assign Project + Priority + Due to anything you captured loosely during the week. Open Projects → Active Projects board → walk each card. Open Tasks → Waiting On → poke anyone you're blocked by. Open Meetings → 1:1 Log by Person → skim each report's recent themes before next week.

6. The fancy home page — design + how to recreate

The home page should feel like a calm command center, not a wall of databases. Here's the design and the exact blocks you need.

Design principles

  1. Cover image — sets the mood the moment you open it. Pick something atmospheric (mountains, sunrise, fog over water). Avoid busy stock photos.
  2. Big icon — single emoji or custom icon. Mine is 🧠.
  3. Quote at top — one line, italicized, your favorite productivity quote. Anchors the page philosophically.
  4. Welcome callout — one sentence explaining what this page is.
  5. Section headings with emoji — ☀️ Today's Mission, 🧭 The Six Databases, 🔄 The Flow, 🌅 Daily Ritual, 🌙 Weekly Review, 🗝 Cheat Sheet.
  6. Dividers between sections — Notion's --- creates a thin horizontal rule that feels intentional.
  7. Inline checkboxes for the rituals — they reset visually each day and feel actionable.
  8. Closing note — one philosophical reminder at the bottom.

Recreate it block-by-block

1Add a cover. Hover the top of the page → "Add cover" → "Change cover" → Unsplash tab → search for "mountain", "fog", "sunrise". Or upload your own. Reposition until it looks calm.
2Add an icon. Click the top-left icon area → pick an emoji (🧠, 🌌, 🧭, ⚡). Make it one consistent emoji that becomes "yours."
3Type your title: Productivity OS. Don't add the emoji here — the icon does that.
4Quote block. Press /quote → Enter. Paste your favorite productivity quote in italics. End with em-dash and the author.
5Welcome callout. Press /callout → Enter. Pick a 💡 or 🧠 emoji. Type one sentence describing what this page is.
6Divider. Press /divider → Enter. Use these between every major section.
7Section heading. Type ## ☀️ Today's Mission (the ## auto-converts to H2). Always lead with an emoji — it makes scanning easy.
8Tip callout. Press /callout → 🎯 emoji → "Start here every morning…"
9Six databases section. Heading: ## 🧭 The Six Databases. Then drag each database from the sidebar into the page (or press /linked → "Linked view of database" if you want them collapsible). They'll appear as full database blocks. Tip: if you want them to feel more like a menu and less like 6 huge tables, use /linked and pick "List" type for each.
10The Flow section. Heading: ## 🔄 The Flow. Numbered list explaining the 3-step capture flow. Add a /callout at the bottom marked ✅ saying "The linchpin is the Tasks → Meeting relation."
11Daily Ritual. Heading: ## 🌅 Daily Ritual (5 minutes). Press /todo → write checkboxes for each step. They visually reset each morning.
12Weekly Review. Same pattern: ## 🌙 Weekly Review (Friday, 30 min) + /todo checkboxes.
13Cheat sheet table. Heading: ## 🗝 Key Views Cheat Sheet. Press /table → Simple table → 3 columns: Database / View / When to use. Fill in the 8 views.
14Closing note. Final divider, then a /callout with 💡: "Remember: the goal isn't to use every feature. It's to write each thing exactly once."
Polish tips:
  • Use /columns to put Daily Ritual and Weekly Review side-by-side if your screen is wide.
  • Hover any block → ⋮⋮ → Color → pick a soft background tint (gray, brown, default) to group sections.
  • Press Cmd/Ctrl + Shift + L to toggle dark mode and make sure the cover still works.
  • For a "fancier" look, replace the section headings with toggle headings (/toggle h2) so each section collapses — turns the page into a clean menu.

7. Step-by-step replication from scratch

If you're rebuilding this in your Affirm work Notion (or anywhere fresh), here is the exact sequence. Do them in order — relations require their target databases to already exist.

1Create the hub page. New page → title: Productivity OS. Add cover + 🧠 icon.
2Create People (no relations yet). Inside the hub: /database - inline. Title = "People". Properties:
  • Name (title)
  • Relationship (select: Manager, Direct Report, Peer, Stakeholder, External)
  • Role (text)
  • Team (text)
  • Email (email)
3Create Areas. Properties: Area (title), Type (select: Responsibility, Goal, Resource), Description (text).
4Create Projects. Properties: Project (title), Status (select: Backlog, Planning, In Progress, Blocked, Done, Archived), Priority (select: P0–P3), Goal (text), Start (date), Target (date), PRJ (auto-increment ID). Now add relations: → Areas, → People (rename to "Stakeholders").
5Create Meetings & 1:1s. Properties: Title (title), Type (select: 1:1 Manager, 1:1 Direct Report, Project Meeting, Team Sync, Stakeholder, Other), Date (date), Summary (text). Relations: → People (rename to "Attendees"), → Projects.
6Create Tasks. Properties: Task (title), Status (select: Inbox, Next, In Progress, Waiting, Done), Priority (P0–P3), Context (multi-select: @work, @home, @deep, @quick, @waiting), Do Date (date), Due (date), TSK (auto ID). Relations (the linchpin step): → Projects, → Meetings, → People (rename to "Assignee").
7Create Notes & Knowledge. Properties: Title (title), Type (select), Tags (multi-select), Created (date). Relations: → Projects, → Meetings.
8Create the views (see table in section 4). For each one: open the database → click "+ New view" → pick the type → set the filter/sort/grouping shown.
9Create the 3 meeting templates. Open Meetings → "+ New" dropdown → "+ New template". Make three: 1:1 with Manager, 1:1 with Direct Report, Project Meeting. Use the headings shown in section 9 below. Set the Type property in each template so it auto-fills.
10Build the home page using section 6 above.
11Seed it with reality — add your manager, your reports, your top 2–3 active projects, and the next 1:1s on your calendar. Don't try to backfill everything. Just start using it for new stuff.
